# 一、优先排查（90%概率问题）

## 1️⃣ 光伏上网量统计被重复计算

你这里有一个很明显的统计问题：

```csharp
// PV统计
stats.TotalGridExport += result.PVToGrid;
```

后面 **电网统计又统计了一次**

```csharp
if (result.GridPower < 0)
{
    stats.TotalGridExport += Math.Abs(result.GridPower);
}
```

这意味着：

**PVToGrid + GridPower<0 可能是同一份电量**

结果：

```
TotalGridExport = PVToGrid + GridExport
```

如果：

```
PVToGrid == -GridPower
```

那就 **统计翻倍**

当储能容量变大时：

- 上网电量增加
- 收益被 **翻倍放大**

可能就会出现：

```
现金流 1400w → 1亿
```

### 建议

只保留一个：

```
TotalGridExport = abs(GridPower)
```

或者：

```
TotalGridExport = PVToGrid
```

# 二、第二高概率问题

## 2️⃣ SOC单位严重混乱

你的代码：

```
SOCMin => (1 - DOD) * RatedCapacity
SOCMax => RatedCapacity
```

说明：

```
SOC 单位 = kWh
```

但初始化：

```
_currentSOC = essParams.InitialSOC;
```

参数注释：

```
InitialSOC (kWh)
```

但是更新SOC：

```csharp
_currentSOC += (-essPower) * efficiency / RatedCapacity;
```

这里变成：

```
kW / kWh
```

单位变成：

```
SOC = ratio
```

而 SOCMin / SOCMax 是 **kWh**

这会导致：

```
SOC比较完全错误
```

例如：

```
SOCMax = 1000
SOC = 0.5
```

判断：

```
SOC < SOCMax
```

永远成立。

结果：

**储能永远可以充电**

容量越大：

```
充电能力越无限
```

然后：

```
套利收益爆炸
```

这是一个 **非常严重的 bug**。
